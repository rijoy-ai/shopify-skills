# Review themes → page structure playbook

For `review-trust-page-optimization`.

---

## 1. Common clusters → structural fixes

| Cluster | Typical barrier | Page fixes | Trust module |
|---------|-----------------|------------|--------------|
| Runs small/large | Size expectation | Size guide above fold; model stats; "runs small" label | Fit survey quote; exchange policy for size |
| Color different | Screen vs real | Swatches + lighting note; customer photos | "Colors may vary by display" + real UGC |
| Quality / flimsy | Overpromise in hero | Soften superlatives; close-up texture; weight spec | Material spec + warranty scope |
| Not as described | Missing detail | Bullets match review gaps; what's in box | Spec table |
| Late / damaged | Post-purchase fear | Shipping SLA + pack photo on PDP | On-time ship badge if true |

---

## 2. Selling-point correction (logic)

1. **Audit hero claim** vs top 1-star themes — conflict = rewrite or relocate.
2. **Front-load warnings** that reviews repeat — reduces wrong buyers, can lift **qualified** CVR.
3. **Replace vague trust** ("#1 rated") with **evidence** (return rate down after fit note, star breakdown).

---

## 3. Trust badge refactor rules

- **Remove** badges that reviews debunk (e.g. "premium" if many "feels cheap").
- **Add** claim-aligned modules: verified dimensions, fit recommendation, shade matcher link.
- **Order**: expectation-setting **before** aggressive discount urgency.

---

## 4. Not this skill’s job

- Bulk polite replies without page changes (negative boundary).
- Deleting or hiding legitimate reviews (unethical / platform policy).

# Shopify Ops SOPs (Routines + KPIs)

把“运营经验”变成可复制的 SOP：**每日/每周/每月例行 + 关键报表口径 + 任务分工 + 质量门槛**。不绑定任何 App。

## 输入（缺失就追问）

1. 团队结构：谁负责（owner / marketing / support / ops / content）
2. SKU 数量与上新节奏
3. 主要国家与时区
4. 客服渠道与 SLA（邮件/工单/社媒）
5. 复购周期与促销策略（是否订阅、是否能常打折）

## 输出格式（固定）

```markdown
## Operating system overview
- Cadence:
- Roles:
- Rituals:

## SOP library (copy/paste)
### SOP: <name>
- Owner:
- Frequency:
- Inputs:
- Steps:
- Done definition:
- KPIs:
- Templates:

## KPI glossary (definitions)
- ...

## Weekly review agenda (60 minutes)
- ...
```

## 必备 SOP（至少产出 8 个）

从下面挑选并按商家情况定制：

### Retention / Lifecycle
- Post-purchase content QA（发什么、何时发、如何迭代）
- Replenishment & win-back calendar（按复购周期维护）
- Offer governance（折扣使用规则、毛利红线、白名单人群）

### Merchandising
- PDP QA checklist（首屏价值主张、政策、FAQ、变体）
- Collection hygiene（筛选器、排序逻辑、断货处理）
- Bundle strategy（套装定义、上架、库存联动）

### Support (churn prevention)
- Low-rating / complaint triage（优先级、补救、升级路径）
- Refund prevention script（关键信息收集、替代方案、补发规则）

### Content / Social proof
- UGC intake & reuse（授权、素材库、复用到 PDP/email 的流程）
- Reviews response workflow（回复话术、负评处理、产品反馈闭环）

## KPI 口径（必须给“怎么计算”）

至少包含：

- Revenue, orders, AOV
- CVR（按 device）
- Repeat purchase rate（30/60/90）
- Time-to-2nd-order
- Refund rate / chargeback rate
- Contribution margin（促销后）

## 输出规则

- 每个 SOP 必须有“Done definition”（什么叫做完，避免空转）
- 每个 SOP 至少绑定 1 个 KPI
- 如果用户说“我们人手少”：给出“最小可行 SOP 组合”（3 个）与自动化建议
